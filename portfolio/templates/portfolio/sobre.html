Esquema da Arquitetura

Componentes e Fluxo de Informação

1. Browser & URL Dispatcher

Tudo começa com um HTTP Request enviado pelo browser. O urls.py faz o URL match para saber qual a rota pedida e passa o pedido pela Middleware Stack.

2. View (O Controlador Central)

O views.py recebe o pedido. Se houver formulários (forms.py), processa a form data. De seguida, faz uma query ao Model para obter os dados necessários e prepara o Context para enviar ao Template.

3. Model & Base de Dados

O models.py interage com a Django ORM. A QuerySet API converte os pedidos Python em SQL para ler ou gravar na Database (SQLite). O Django Admin também interage diretamente aqui fazendo operações CRUD.

4. Template Engine & Response

O Template Engine (DTL) recebe o Contexto da View, renderiza o ficheiro HTML e injeta os Static Files. O resultado final volta pela View e é entregue ao Browser como um HTTP Response.

Vantagens no Desenvolvimento de Software:
  • Histórico Seguro: Permite reverter o código para estados anteriores caso ocorram erros, garantindo que nada se perde.
  • Colaboração e Organização: Facilita o trabalho em equipa e a experimentação segura através de branches (ramificações) sem interferir com o código principal.
  • Acessibilidade e Integração: O código fica protegido na cloud e acessível em qualquer lugar, facilitando também futuras implementações de CI/CD (Integração e Entrega Contínuas).
